# zmux Keybindings — Source of Truth

> **Generated** from `internal/keys` by `zmux keys gen`. Do not edit by hand —
> change the registry and run `zmux keys gen` (CI fails on drift).

zmux owns three keybinding tables it generates into `tmux.conf` (prefix,
no-prefix, copy-mode) plus a set of inherited tmux defaults it deliberately
keeps. The tables below are rendered from the registry.

## TUI surfaces

Component-local keys for the interactive surfaces. These keys live in their
surface packages (`internal/tui/...`), not the shared registry, because they are
scoped to a single Bubble Tea model.

### Picker (workspace picker — outside tmux or popup)

Single flat-list model: top action row + workspaces + nested sessions (when
expanded). Text input is always active, so destructive actions use `ctrl+`
modifiers.

| Action | Key | Notes |
|--------|-----|-------|
| `select` | `enter` | Create / drill in / attach, depending on row |
| `quit` / `cancel` | `esc` (empty) / `ctrl+c` | Esc with text clears query first |
| `kill` | `ctrl+x` | Delete workspace or session under cursor (confirms) |
| `toggle.empty` | `ctrl+h` | Show / hide workspaces with no live sessions |
| `accept.suggestion` | `tab` | Accept ghost autocompletion (workspace name) |
| `navigate.up` / `down` | `up` / `down` | Tree traversal |
| `session.goto.N` | `1-9` | Quick-select session by index (when input empty) |

**Search grammar**: typing fuzzy-matches workspace names. A space separator
filters sessions: `myapp dev` shows sessions matching `dev` within workspaces
matching `myapp`.

## Design rules

1. **`x` always means kill/close/stop** — `ctrl+x` in pickers, `x` in dashboards, `prefix+x` in tmux.
2. **`d` always means detach** — never kill/delete.
3. **`enter` always means confirm/select.**
4. **`esc` always means cancel/back/quit.**
5. **Pickers use `ctrl+` modifiers** because text input is always active.
6. **Dashboard tabs use single letters** because the popup captures input.
7. **Prefix keys are for tmux-level ops** (tabs, sessions, config).
8. **No-prefix `Alt+` keys are instant** — direct tab/pane actions; `prefix + Alt+1-9` is the matching session jump.
9. **Context resolves same-key conflicts** — `n` = new (dashboard) vs next tab (prefix); `r` = rename (dashboard) vs reload (prefix). Different input contexts mean no real conflict.

## Collision notes for terminal / WM / editor

- **Ghostty:** `Alt+Shift+Arrow`, `Alt+Arrow`, and `Alt+1-9` are unbound locally so they pass through to tmux/zmux. `Ctrl+V` is also unbound so copy-mode rectangle selection works.
- **Hyprland:** uses `Super+Arrow` / `Super+Shift+Arrow` / `Super+Ctrl+Arrow` for window focus/move/resize — these do not consume tmux prefix sequences or no-prefix `Alt+Shift+Arrow`.
- **Pane arrows:** no-prefix `Alt+Shift+Arrow` focuses panes; `prefix + Ctrl+Arrow` / `prefix + Alt+Arrow` resize panes in tmux's prefix table. Keeping resize behind prefix avoids stealing editor `Ctrl+Arrow` and shell `Alt+Arrow` behavior.
- **Neovim/LazyVim:** `Ctrl+h/j/k/l` and `Ctrl+Arrow` are used for window navigation/resize. zmux avoids global no-prefix `Ctrl+h/j/k/l` pane bindings by default; use tmux prefix pane keys instead.
