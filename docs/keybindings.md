### Inherited tmux pane keys

zmux keeps tmux's useful pane defaults active instead of reinventing them.
These are inherited tmux bindings, not custom zmux bindings, but they are part
of the supported workflow and matter for sidecar use.

| Action | Key | Notes |
|--------|-----|-------|
| `pane.focus.left/right/up/down` | `prefix + ←/→/↑/↓` | Move focus between panes |
| `pane.next` | `prefix + o` | Cycle to next pane |
| `pane.last` | `prefix + ;` | Return to previous pane |
| `pane.ids` | `prefix + q` | Show pane numbers/ids overlay |
| `pane.zoom` | `prefix + z` | Toggle zoom for current pane |
| `pane.resize.small` | `prefix + Ctrl+←/→/↑/↓` | Resize by one cell |
| `pane.resize.large` | `prefix + Alt+←/→/↑/↓` | Resize by five cells |
| `pane.split.right` | `prefix + %` | Split current pane right |
| `pane.split.down` | `prefix + "` | Split current pane below |
| `pane.swap.up/down` | `prefix + {` / `prefix + }` | Swap pane order |
| `pane.break` | `prefix + !` | Break pane into its own window |

Auto-named tabs normally use tmux's command name (`pi`, `bash`, etc.). When
multiple tabs in the same session share that name, zmux marks them as
`name[cwd]` in the bar (for example `pi[zmux]`) with the cwd suffix dimmed.
Mouse support is enabled: click focuses panes, dragging borders resizes panes,
and right-clicking a pane opens tmux's native pane menu. Failed or signalled
foreground commands stay visible as dead panes, preventing Ctrl+C spam from
silently deleting the tab; clean exits close normally. Use `prefix+x` /
`zmux tab kill` for intentional stopped-tab closure. Pane border headers are
enabled only for split windows: the active pane shows id, title, command, size,
and the `A-S arrows` focus hint; inactive panes show a calmer index/title label.
Split indicators and active-border color make focus visible around dividers and
bottom edges while pane backgrounds stay transparent/default. Single-pane
windows keep the border header blank.

### Collision notes for terminal / WM / editor

- **Ghostty:** `Alt+Shift+Arrow`, `Alt+Arrow`, and `Alt+1-5` are unbound in
  the local Ghostty config so they pass through to tmux/zmux. zmux uses
  `Alt+Shift+Arrow` for instant pane focus. `Ctrl+V` is also unbound so
  copy-mode rectangle selection can work.
- **Hyprland:** current Hyprland bindings use `Super+Arrow`,
  `Super+Shift+Arrow`, and `Super+Ctrl+Arrow` for window focus/move/resize.
  They do not consume plain tmux prefix sequences or `Alt+Shift+Arrow`.
- **Neovim/LazyVim:** `Ctrl+h/j/k/l` and `Ctrl+Arrow` are used for Neovim
  window navigation/resize. zmux should avoid global no-prefix `Ctrl+h/j/k/l`
  pane bindings by default; use tmux prefix pane keys or an explicit optional
  vim-tmux-navigator-style integration later.

### Picker (workspace picker — outside tmux or popup)

Single flat-list model: top action row + workspaces + nested sessions (when expanded).
Text input always active → actions use `ctrl+` modifiers.

| Action | Key | Notes |
|--------|-----|-------|
| `select` (top, empty) | `enter` | Create tmp-N session (no workspace) |
| `select` (top, typed) | `enter` | Create workspace named `<query>` + main session |
| `select` (workspace) | `enter` | Drill into sessions, or create default session if empty |
| `select` (session) | `enter` | Attach to that session |
| `quit` / `cancel` | `esc` (empty) / `ctrl+c` | Esc with text clears query first |
| `kill` | `ctrl+x` | Delete workspace or session under cursor (confirms) |
| `template` | `ctrl+t` | Open template picker |
| `toggle.empty` | `ctrl+h` | Show / hide workspaces with no live sessions |
| `accept.suggestion` | `tab` | Accept ghost autocompletion (workspace name) |
| `navigate.up` | `up` | Tree traversal (workspaces + visible sessions) |
| `navigate.down` | `down` | Tree traversal |
| `session.goto.N` | `1-9` | Quick-select session by 1-based index (when input empty) |

**Search grammar**: typing fuzzy-matches workspace names. A space-separator
filters sessions: `myapp dev` shows sessions matching `dev` within workspaces
matching `myapp`. Matched characters are underlined in workspace names.

**Expansion**: a workspace's sessions show inline beneath it when the cursor
is on the workspace (or on one of its sessions), OR when there's an active
search and the workspace matches.

## Design Rules

1. **`x` always means kill/close/stop** — `ctrl+x` in pickers, `x` in dashboards, prefix+`x` in tmux
2. **`d` always means detach** — never kill/delete
3. **`enter` always means confirm/select**
4. **`esc` always means cancel/back/quit**
5. **Pickers use `ctrl+` modifiers** because text input is always active
6. **Dashboard tabs use single letters** because the popup captures input
7. **Prefix keys are for tmux-level ops** (tabs, sessions, config)
8. **`Alt+` keys are instant** — no prefix needed
9. **Context resolves same-key conflicts** — `n` = new (dashboard) vs next tab (prefix), `r` = rename (dashboard) vs reload (prefix). Different input contexts = no real conflict.
